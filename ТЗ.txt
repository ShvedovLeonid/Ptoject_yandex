Техническое задание на разработку игры "Arcade Doodle Jump"

1. Общее описание проекта

Проект: Аркадная игра-платформер в стиле Doodle Jump с использованием библиотеки Arcade на Python. Цель игрока — прыгать по платформам, поднимаясь как можно выше, избегая падения. Игра демонстрирует практическое применение изученных технологий.

2. Технические требования и применяемые технологии

2.1 Управление зависимостями

Проект должен использовать файл requirements.txt для четкого определения всех необходимых пакетов Python, включая arcade и дополнительные библиотеки для работы с файлами и звуком.

2.2 Структура приложения

Финальное окно: Экран завершения игры с отображением итогового счета, кнопкой перезапуска.

Подсчет и вывод результатов: Система подсчета очков в реальном времени с сохранением результатов.

2.3 Графика и визуализация

Спрайты: Все игровые объекты реализованы через спрайты с отдельными текстурами.

Камера: Динамическая камера, следующая за игроком вверх, с плавным перемещением.

2.4 Игровая физика и взаимодействия

Физический движок: Использование встроенных возможностей Arcade для физики (гравитация, столкновения, импульсы).

Столкновения: Система обнаружения столкновений между всеми типами игровых объектов.

2.5 Аудио и данные

Звук: Фоновая музыка.

Хранение данных: Сохранение рекордов в файлах.

2.6 Уровни и контент

Несколько уровней: Минимум 2 уровня с разными наборами платформ и сложностью.

Прогрессия: Увеличение сложности по мере прохождения уровней.

3. Игровая механика

Игрок управляет персонажем, который автоматически прыгает при приземлении на платформы. Управление горизонтальным движением осуществляется стрелками или WASD. Камера следует за персонажем вверх, создавая бесконечный вертикальный скроллинг. При падении ниже определенной точки игра заканчивается.

4. Архитектура приложения

Проект построен по модульному принципу с четким разделением ответственности. Основные модули включают управление игровым циклом, обработку ввода, физические вычисления, рендеринг графики, управление звуком и работу с данными. Используется объектно-ориентированный подход с наследованием для типов игровых объектов.
